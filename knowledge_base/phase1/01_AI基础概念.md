# AI 基础概念

## 什么是人工智能（AI）
人工智能就是让计算机具备类似人类的感知、推理、学习和决策能力的技术总称。它不是一个具体的"东西"，而是一大类技术的统称。

发展三阶段：
- **规则时代（1950s-1990s）**：人工写规则，if-else 逻辑。如早期象棋程序。
- **机器学习时代（1990s-2010s）**：从数据中学习规律，不再手写规则。如垃圾邮件分类。
- **深度学习/大模型时代（2010s-至今）**：用海量数据训练超大神经网络，涌现出通用能力。

## 什么是大语言模型（LLM）
大语言模型 = 一个经过海量文本训练的"下一个词预测器"。

类比：你打字时输入法会猜你下一个字要打什么。LLM 就是这个猜词能力放大到极致——用数千亿参数、数万亿字的语料训练，使得它"猜"出来的内容逻辑通顺、知识丰富。

- **参数**：模型内部可调整的"旋钮"数量。参数越多，模型越"聪明"，但也越贵越慢。
- **Token**：模型处理文本的最小单位。一个中文汉字≈1个Token，一个英文单词≈1-2个Token。
- **上下文窗口**：模型一次能"看到"的文本上限。200K上下文≈能一次处理一本中等厚度的书。
- **温度（Temperature）**：控制输出随机性的参数。0=确定性强（适合编程），1=创造力强（适合写作）。

## 训练 vs 推理
- **训练** = 学生上学读书。耗时数月/数年，消耗海量算力（数千万美元），得到"学会了知识"的模型。
- **推理** = 学生考试答题。每次对话就是一次推理，速度快（秒级），成本低（按Token计费）。

## 什么是 AI Agent
**AI Agent = 大语言模型 + 工具调用能力 + 自主规划与决策循环**

类比理解：
- LLM = 只有大脑的人（会思考但手脚不能动）
- Agent = 有大脑 + 有手脚 + 会用工具的人（会思考也能做事）

Agent 工作循环：
```
感知环境 → 思考分析 → 决策行动 → 观察结果 → 感知环境 → ...
```

Agent 的核心组件：
1. **LLM（大脑）**：理解任务、拆解步骤、判断何时结束
2. **工具（Tool）**：搜索、读文件、执行命令、操作数据库等
3. **规划（Planning）**：将复杂任务拆解为有序步骤
4. **记忆（Memory）**：短期记忆（当前对话上下文）+ 长期记忆（知识库/向量数据库）

举例：你让 Agent "帮我查一下明天北京的天气，如果下雨就发邮件提醒我带伞"——Agent 会自主调用天气API→判断是否下雨→如果下雨调用邮件API→完成。

## 什么是 MCP（模型上下文协议）
MCP = Model Context Protocol，Anthropic 发布的开放标准协议。

解决的问题：以前每个 AI 工具都要单独对接各种外部服务（搜索API、数据库、文件系统），接口各不统一。MCP 提供了一套统一的标准接口。

类比：MCP 就像 USB-C 接口——以前手机、电脑、耳机各有各的充电口，USB-C 统一了所有接口。

MCP Server 提供三种资源：
- **Tools**：AI 可以调用的动作（搜索网页、读取文件、发邮件）
- **Resources**：AI 可以读取的数据（文档、数据库记录、配置）
- **Prompts**：预定义的提示词模板（方便复用）

## 什么是 RAG（检索增强生成）
RAG = 先从知识库中检索相关信息，再把这些信息注入到 Prompt 里让 LLM 生成回答。

类比：期末考试开卷考 vs 闭卷考。RAG 就是让 LLM "开卷考试"——可以翻书（检索知识库）再作答。
