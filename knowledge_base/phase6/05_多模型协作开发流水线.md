# 多模型协作开发流水线

## 核心理念
不同模型各司其职，就像公司里不同岗位：
- CEO（规划）：GPT-5.4 Pro，负责战略设计
- 工程师（执行）：Claude Code + 性价比模型，负责干活
- QA（验收）：监督 Agent，负责质检

## 三阶段流水线

### 阶段1：规划（Planning）
- **执行者**：GPT-5.4 Pro / Claude Opus
- **平台**：Cherry Studio / GPT 官网
- **产出**：完整项目规格书（spec）——字段设计、接口定义、架构方案
- **关键**：这个阶段只出方案，不写一行代码

### 阶段2：编码（Coding）
- **执行者**：Claude Code CLI + 性价比模型（DeepSeek/GLM 5.1/Kimi K2.6/GPT 5.3 Codex）
- **平台**：终端 + MultiCA（多Agent看板工具）
- **方式**：任务自动拆分 → 多 team 并发执行
- **关键**：任务必须切得足够细，保持每个子任务的上下文清晰

### 阶段3：验收（Review）
- **执行者**：专门的监督型 Agent（配置 Claude 等强模型）
- **方式**：每个 Phase 完成后独立验收
- **检查项**：代码是否符合 spec、逻辑是否通顺、是否偏离需求
- **关键**：必须逐 Phase 验收，不能等全部写完再看

## 核心原则（来自社区最佳实践）
1. **任务切细**：细到每个子任务能在一次对话内完成 → 稳定上下文 → 规避幻觉
2. **无监督=失控**：长上下文模型也扛不住无监督长流程，"目标和结果极易漂移"
3. **人工验收最高效**：花时间做 Review 比花时间返工修改成本低得多
4. **模型选对**：GPT 5.3 Codex Medium、Kimi K2.6 这些中端模型在编码场景已足够

## 工具推荐
- **MultiCA**：统一管理 Codex 和 Claude Code 的看板工具，类似 Vibe Kanban
- **Harness**：元技能，用 AI 帮你设计 Agent 团队结构

## 新手简化版
如果觉得三阶段太复杂，先从两阶段开始：
1. **规划+编码**：用 Claude Code 直接写
2. **验收**：把代码贴到 Cherry Studio 另一个对话，让 AI 做 Code Review
