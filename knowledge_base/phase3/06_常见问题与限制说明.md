# 第三方模型接入限制说明

## Claude Code 中第三方模型的已知限制
1. **Vision/多模态**：DeepSeek 等纯文本模型无法识别图片，报错截图的解决方案见 09_ 文件
2. **Tool Use 能力差异**：第三方 tool_use 可能不如 Claude 原生稳定
3. **Agent Skills 兼容性**：部分 Skills 依赖 Claude 原生能力，第三方模型可能降级
4. **Prompt Caching**：缓存机制可能不适用于第三方模型
5. **/compact 行为差异**：压缩上下文的效果可能不同

## 功能对照表
| 功能 | Claude 原生 | DeepSeek | GPT |
|------|-----------|----------|-----|
| 代码生成 | ✅ | ✅ | ✅ |
| 工具调用 | ✅ | ⚠️ 有限 | ✅ |
| 图片分析 | ✅ | ❌ | ✅ |
| Prompt 缓存 | ✅ | ❌ | ⚠️ |
| Agent Skills | ✅ | ⚠️ 部分 | ⚠️ 部分 |

## 建议
- 核心开发任务用 Claude 原生
- 高频简单任务用 DeepSeek 替代
- 重要项目上线前用 Claude 做最终验证
