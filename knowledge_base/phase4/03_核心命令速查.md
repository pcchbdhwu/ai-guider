# Git 核心命令速查

## 基础操作
| 命令 | 作用 |
|------|------|
| `git init` | 初始化本地仓库 |
| `git clone <url>` | 克隆远程仓库到本地 |
| `git status` | 查看当前状态（改了什么） |
| `git add <file>` | 暂存文件（`git add .` 暂存全部） |
| `git commit -m "信息"` | 提交到本地仓库 |
| `git push` | 推送到远程 |
| `git pull` | 拉取远程更新 |
| `git log --oneline` | 查看提交历史（简洁版） |
| `git diff` | 查看具体改了什么 |

## 分支操作
| 命令 | 作用 |
|------|------|
| `git branch` | 查看所有分支 |
| `git branch <name>` | 创建新分支 |
| `git checkout <name>` | 切换到某分支 |
| `git checkout -b <name>` | 创建并切换到新分支 |
| `git merge <name>` | 合并某分支到当前分支 |
| `git branch -d <name>` | 删除已合并的分支 |

## 撤销与回退
| 命令 | 作用 | 危险性 |
|------|------|--------|
| `git restore <file>` | 撤销工作区修改 | 低 |
| `git restore --staged <file>` | 取消暂存 | 低 |
| `git reset --soft HEAD~1` | 撤销最近一次commit（保留改动） | 中 |
| `git reset --hard HEAD~1` | 彻底回退（丢弃改动） | ⚠️ 高 |

## 工作流速记
```bash
# 日常开发
git pull              # 1. 拉最新代码
# ... 改代码 ...
git add .             # 2. 暂存所有改动
git commit -m "做了什么"  # 3. 提交
git push              # 4. 推送

# 分支协作
git checkout -b feature/xxx  # 1. 新建功能分支
# ... 开发 ...
git add . && git commit -m "xxx"
git push -u origin feature/xxx  # 2. 推送分支
# 3. 到 GitHub 网页发 PR
```
