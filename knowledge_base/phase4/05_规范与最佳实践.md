# Git Commit 规范与最佳实践

## Commit Message 规范
```
<type>(<scope>): <简短描述>

<详细描述>
```

**类型（type）：**
| 前缀 | 含义 | 示例 |
|------|------|------|
| feat | 新功能 | feat: 添加用户登录功能 |
| fix | 修复Bug | fix: 修复登录超时未重定向的问题 |
| docs | 文档修改 | docs: 更新安装指南 |
| style | 代码格式（不影响功能） | style: 统一缩进格式 |
| refactor | 重构（不增功能不修Bug） | refactor: 提取公共验证逻辑 |
| test | 测试相关 | test: 添加登录模块单元测试 |
| chore | 杂项（依赖/配置等） | chore: 更新依赖版本 |

## .gitignore 模板
```gitignore
# 依赖
node_modules/
__pycache__/
*.pyc

# 环境变量/密钥（重要！）
.env
.env.local
*.pem
secrets/

# IDE
.vscode/
.idea/

# 系统文件
.DS_Store
Thumbs.db
```

## 开源协议选择
| 协议 | 特点 | 适用 |
|------|------|------|
| MIT | 最宽松，随便用 | 想让别人随便用 |
| GPL | 传染性，用了必须开源 | 强制开源 |
| Apache 2.0 | 有专利授权条款 | 企业级项目 |

## PR 模板
```markdown
## 改动内容
简述改了什么

## 改动原因
为什么这样改

## 测试情况
- [ ] 本地测试通过
- [ ] 相关测试用例通过

## 截图（如有UI变化）
```
