# Git 基础理念

## 版本控制是什么
版本控制 = 文件的"历史记录 + 时光机"。它能记录每次修改、谁改了什么、随时可以回退。

## 集中式 vs 分布式
- **集中式（SVN）**：只有一个中央仓库，断网不能工作
- **分布式（Git）**：每个人本地都有完整仓库，断网也能提交，联网后同步

## Git 四区模型
```
工作区(Working) → 暂存区(Staging) → 本地仓库(Local) → 远程仓库(Remote)
   (改文件)          (git add)        (git commit)      (git push)
                                                    ←  (git pull)
```
- **工作区**：你正在编辑的文件
- **暂存区**：标记为"下次提交要包含"的文件
- **本地仓库**：已提交的历史版本
- **远程仓库**：GitHub/Gitee 上的云端版本

## 核心概念
- **Commit（提交）**：一次快照，记录当时的文件状态
- **Branch（分支）**：独立的开发线，彼此隔离
- **Merge（合并）**：把分支的改动合并到一起
- **Conflict（冲突）**：两个分支改了同一个地方，需人工裁决
